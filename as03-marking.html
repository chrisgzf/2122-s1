    As03 marking

    • warning: implicit conversion loses floating-point precision: 'double' to 'float' [-Wconversion]

    -1 per question.

    Sample Feedback:

    float is not precise enough for most practical purposes. Use only double. Converting from double to float causes a loss of precision.

    • warning: implicit conversion loses floating-point precision: 'double' to 'float' [-Wconversion]

    -1 per question.

    Sample Feedback:

    float is not precise enough for most practical purposes. Use only double. Converting from double to float causes a loss of precision. (-1)

    • warning cast from function call of type 'double' to non-matching type 'long'

    Likely a bug or using the wrong choice of type / function. -1 if it is a bug, -1 per question if it is wrong choice of type.

    • warning: type specifier missing, defaults to 'int' [-Wimplicit-int]

    This is ok only for main. If no return type is declared for any other functions, -1 for each question.

    Sample Feedback:

    You need to declare the return type of your functions (-1).

    • warning: implicit conversion loses integer precision: 'long' to 'int' [-Wshorten-64-to-32]

    If there is no implicit constraint on the semantic of the data (e.g., month, year, day), long should be used. -1 for each question.

    Sample feedback:

    Please use long for integer type. int is guaranteed to be at least 16 bits only, too limited for many purposes. (-1)

    • void function 'circle' should not return void expression [-Wpedantic]

    Given them a warning but no mark deduction.

    Sample feedback:

    You should not return anything from a void function. This return is redundant and should be removed. - warning: loop will run at most once (loop increment never executed)

    Possibly buggy loop, or students do not understand the concept of loop. -1 for each occurence.

    Sample Feedback:

    Your loop will run at most once in this case. There is no need to use a loop. (-1)

    • warning: macro is not used [-Wunused-macros]

    Just point out to the students. No penalty.

    • warning: implicit conversion changes signedness: 'long' to 'unsigned long' [-Wsign-conversion]

    Please warn the student (no penalty)

    Sample Feedback:

    Mixing signed and unsigned variables is dangerous. Beginners should use only signed variables.

    • warning: possible misuse of comma operator here [-Wcomma]

    No penalty

    Sample Feedback:

    You should avoid using the , operator, which makes your code harder to read.

    Style

    There is 2 style marks for each question. We won't penalize for every violation, but instead, judge yourself whether the code is clean, understandable, properly indented, use proper variable names, etc. You can give 0 for very bad style, 1.0 or 1.5 for something in the middle, 2 for ok code. Please give feedback if you find the code hard to read and is violating the guideline.

    Other Issues

    Lots of duplicated code

    -0 mark

    Suggest to them how they can improve their code by factoring out the common parts into functions.

    Didn't include @author / @group

    -1 mark

    Bugs

    In general, -1 for each bug.

    Extra text or newline printed to screen

    -1 for each program

    Return non-zero from main()

    -1

    Even if the program exits successfully.
